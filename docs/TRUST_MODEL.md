# Trust Model

## Core Reading

Loophole is a drafting and pressure-testing aid.

It should be trusted to:

- preserve a visible artifact trail
- keep the human in the loop
- surface candidate failures in a draft rule system
- make revisions and escalations explicit

It should not be trusted to:

- prove consistency
- guarantee exhaustive attack coverage
- determine moral truth autonomously
- convert escalation into evidence of deep principle conflict by itself

## Authority Chain

1. Human principles
2. Human escalated decisions
3. Current legal code
4. Resolved case history
5. Model outputs

Model outputs are advisory and operational, not sovereign.

## Auto-Resolution Policy

Auto-resolution is allowed only when:

- the judge marks a case resolvable
- the legislator returns a parseable legal-code revision
- the revised code survives validation against resolved cases

If any of those fail, the case should not be treated as safely settled.

## Escalation Meaning

Escalation is a contestability mechanism.

It means the current loop could not confidently absorb the case without human review.

That may reflect:

- principle tension
- ambiguous drafting
- validator conservatism
- malformed model output
- regression risk
- model misunderstanding

It is therefore a review path, not a proof of profundity.

## Accepted Output

An accepted revision is:

- written into the current code
- added to the code history
- saved into session artifacts
- treated as precedent for later validator checks

That is stronger than a casual rewrite.
It is still weaker than formal verification.

## Non-Finality

Loophole is intentionally non-final in two ways:

- the legal code remains revisable under later pressure
- the repo’s own claims should remain challengeable as tests, docs, and operator experience improve

Stable artifacts matter, but permanent closure is not the design goal here.

## High-Stakes Exclusion

Do not treat this tool as sufficient for:

- legal advice
- medical or safety policy
- employment or disciplinary decisions
- fully autonomous governance workflows

High-stakes use requires external review, stronger guarantees, and domain-specific controls that this repo does not yet provide.
